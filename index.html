Core Framework

The CIA Framework

Data is a valuable asset. The primary goal of cybersecurity is to maintain three critical properties — collectively known as the CIA Triad — for all data, all the time.

Data is not revealed

Confedentiality

Protection against unauthorized access Only those with proper permission should ever be able to read or view the data

Think of it as a locked diary - only the owner (or those granted access) can read it.

REAL-WORLD BREACH Edward Snowden's 2013 leak of classified NSA government data is a landmark example of a Confidentiality breach.

Data is intact - not modified or corrupted

Integrity

Protection against unauthorized modification. Data mustremain accurate and trustworthy from the moment it is created until it is accessed.

If someone changes your grade in the grade book without permission, that's an Integrity breach.

REAL-WORLD BREACH The Stuxnet worm secretly altered code on Iranian centrifuge machines, causing them to spin too fast and physically break — a classic Integrity attack.

Data is accessible to allowed users

Availability

Protection against denial of service. Authorized users must be able to access data and systems whenever they legitimately need them.

A website that is always down, or an account that stays locked, fails the Availability test.

REAL-WORLD BREACH The Mirai Botnet used 145,000+ infected cameras and devices to flood security journalist Brian Krebs' website with traffic, crashing it for 4 days — a textbook DDoS / Availability breach.

Security Formula: Protection = Prevention + (Detection + Response). Every cybersecurity technique fits into at least one of these three elements, and every element maps back to the CIA Triad.

Protecting Data

The Three States of Data

To choose the right security tool, you need to know what state data is currently in. Data exists in one of three states at any given moment.

Data at Rest

Information stored on hard drives, USB drives, DVDs, or memory — sitting still and not currently being moved or processed.

Example: Saved files on your laptop's hard drive.

Data in Transit

Data actively moving from one system to another — through a network, the internet, or a local file share connection.

Example: Sending an email or uploading a file to the cloud.

Data in Use

Data currently being processed by an application, held in active RAM, or being created/edited by a user right now.

Example: A Word document you have open, or app data loaded in memory.

Different tools protect different states: Encryption for data at rest, secure protocols (HTTPS/TLS) for data in transit, and access controls & authentication for data in use.

The AAA Security Model

Authentication, Access Control & Accounting

The AAA model describes the three tools used in practice to achieve the CIA Triad.

Authentication

Prove who you are. Authentication is the establishment of trust between two parties. Without trust, information cannot safely be shared or accessed.

Access Control

What are you allowed to see or reach? Also called authorization or permissions. Not every authenticated user should access everything.

Accounting

Keep track of what happened. Also called auditing. Logs and records let us review activity, detect anomalies, and respond to incidents.

Something you know (password or PIN) Something you have (smart card, hardware token, code sent to your phone) Something you are (fingerprint, retina scan, voice)

Password Security

Password Attacks & How to Defend Against Them

Passwords are the most common - and most commonly attacked - form of authentication. Understanding the attack types is the first step to building a solid defense.

Brute Force

Method # 1

Software that automatically tests every possible character combination until the correct password is found. Thorough but slow for complex passwords.

Dictionary Attack

Method # 2

Uses a large database of common words, names, and character substitution (like p@ssw0rd) to rapidly guess passwrods. Much faster than pure brute force.

Password Spraying

Method # 3

Tests one weak password against many accounts. Avoids account lockouts because each account lockouts because each account only recieves one incorrect attempt

Credential Stuffing

Method # 4

Uses stolen username/password pairs from a known breach to attack other sites. Effective because 52% of people reuse passwords across accounts.

Something you know (password or PIN) Something you have (smart card, hardware token, code sent to your phone) Something you are (fingerprint, retina scan, voice)

BEYOND PASSWORDS

Methods of Authentication

Modern security goes far beyond passwords. Here are the key authentication methods used in real-world systems - and how combining them creates much stronger protection.

Smart Cards

SOMETHING YOU HAVE

Cards with an embedded chip automatically recognized by an authenticating device. Proximity readers can detect them in a pocket or purse without physical contact.

Digital Certificate

SOMETHING YOU HAVE

Servers use certificates to authenticate with browsers (the padlock in your address bar). Issued by trusted third-party Certificate Authorities (CAs).

Biometrics

SOMETHING YOU ARE

Uses measurable physical characteristics: fingerprint, retina scan, voice or even typing speed. Disney World uses finger biometrics for multi-day pass verification.

Hardware Tokens (OTP)

SOMETHING YOU HAVE

A physical device generates a new One-Time Password (OTP) every 60 seconds. Since each OTP is used exactly once, intercepting it is useless to an attacker

Two-Factor Auth (2FA)

KNOW + HAVE

Combines two factors - such as a password plus a PIN sent to your phone (like Google 2 Step Verificaation) One stolen factor alone is useless

Multi-Factor Auth (MFA)

TWO OR MORE FACTORS

Uses two or more different authentication factors. The gold standard for account security - combinng know, have, and/or are dramatically reduces breach risk

Single Sign-On (SSO)

ONE LOGIN, MANY RESOURCES
